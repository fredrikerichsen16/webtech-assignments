Accessibility and Usability

Our page has decent usability with a simple, modern design and an easy to understand layout. It is responsive with satisfactory design for phone users. The accessibility is good apart from the fact that we forgot to include alt tags on images. The layout is such that it makes sense for screen readers. The input tags have a name tag that indicates their purpose. Each section has a header and the table is easy to understand with a screen reader.

WCAG guidelines most relevant to our page

Guideline 1.1 is important because we have many images on our website that need an alt tag so that people with screen readers can perceive them. An option for increasing text size without breaking the website design would also be good and useful for many people, but it’s difficult to implement. Guideline 1.3 could be adhered to even better. For example the responsiveness of our website is not that good. And the structure could be better and make more sense to screen readers, but there’s a tradeoff between nice visual design and design for screen readers.

Improvement Plan

  • Add alt-tags to images describing the content of the image.
  • Remove the “submit form” part in the printed css as it’s not necessary.
  • Improve the responsiveness.

Principle 1: Perceivable - Information and user interface components must be presentable to users in ways they can perceive.

Guideline 1.1 - Text Alternatives: Provide text alternatives for any non-text content so that it can be changed into other forms people need, such as large print, braille, speech, symbols, or simpler language.

Our website does not adhere to this guideline. To adhere to it we would have to add an alt tag to images with a description of the image. We could also add an option for large print. This is possible as a browser feature, but it would just increase the print size without designing the page around it, so the design would break. For inputs we have used names that describe its purpose, which is the recommended solution.

Guideline 1.3 - Adaptable: Create content that can be presented in different ways (for example simpler layout) without losing information or structure.

Regarding point 1.3.1, the only feature on our website where the structure, relationships, presentation etc. is important is the table. A table’s layout is easily programmatically determined so it is easy to perceive with e.g. a screen reader. Regarding point 1.3.2, the order in which content is presented from left to right corresponds to the order of content in html from top to bottom. For guideline 1.3 we meet the requirements for criteria level A.

Guideline 1.4 - Distinguishable: Make it easier for users to see and hear content including separating foreground from background.

Regarding point 1.4.1, we use color in the banner to separate the “Black Friday Sale” foreground and the blue background. But the background can be any color (we chose blue but can easily change it) and the text is semi-transparent white so it will just be a lighter shade of whichever color the background is. Thus, we don’t use any colors that can’t be distinguished by people with color blindness. Regarding point 1.4.2, we don’t use any audio on our website. For guideline 1.4 we meet the requirements for criteria level A.

Principle 2: Operable - User interface components and navigation must be operable

Guideline 2.1 - Keyboard Accessible: Make all functionality available from a keyboard

Almost no functionality is available from just a keyboard and no mouse, so we do not meet the criteria for level A for this guideline.

Guideline 2.4 - Navigable: Provide ways to help users navigate, find content, and determine where they are

  • 2.4.1: Not relevant to our website as we don’t have a bypass block.
  • 2.4.2: We have titles for each section of the page
  • 2.4.4: The purpose of all our links is obvious. E.g. the navigation bar links are obvious, the “See Deals” link is obvious and the links to different phone brands are obvious.
  • 2.4.3: Our sections are in a nice reasonable order I think.
  • 2.4.6: We have headings for each section and labels for some images.
  • 2.4.8: We only have one page so this is not relevant.
  • 2.4.9: All our link purposes are not that hard to deduce just by the text without context (location).
  • 2.4.10: We use section headings.
