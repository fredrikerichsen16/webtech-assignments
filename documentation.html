Get all

Get all phones from database

  • URL: "/get"
  • HTTP Method(s): GET
  • Success: in the event of a succesful request the response is JSON data in the following format with a status code of 200 (example): 
                [{
                "id": 1,
                "brand": "Apple",
                "model": "iPhone 8",
                "os": "iOS",
                "screensize": 6.5,
                "image": "someurl.jpg"
            }, ...]
  • Error: in the event of an unsuccesful request the response is JSON data with the error object returned by sqlite with a status code of 500 (internal server error).
  • Input: no url query or post data is accepted.

Get one

Get one phone from database by id

  • URL: "/get/:id"
  • HTTP Method(s): GET
  • Success: in the event of a succesful request the response is JSON data in the following format with a status code of 200 (example): 
                {
                "id": 1,
                "brand": "Apple",
                "model": "iPhone 8",
                "os": "iOS",
                "screensize": 6.5,
                "image": "someurl.jpg"
            }
  • Error: in the event of an unsuccesful request the response is JSON data with the error object returned by sqlite with a status code of 500 (internal server error).
  • Input: The url query parameter "id" is required. See URL above for where to place it.

Update

Update one phone in the database

  • URL: "/update"
  • HTTP Method(s): PATCH
  • Success: in the event of a succesful request the response is JSON data in the following format with a status code of 200 (example): 
                { 
                success: true, 
                message: "Succesful update on phone with id 1"
            }
  • Error: in the event of an unsuccesful request the response is JSON data with the error object returned by sqlite with a status code of 500 (internal server error).
  • Input: No url query parameter is expected. Post data is expected. "id" is required to indicate which phone to update, as well as any of the column names and the new value to update with. I.e. "brand", "model", "os", "screensize" and "image".

Delete

Delete one phone in the database

  • URL: "/delete/:id"
  • HTTP Method(s): DELETE
  • Success: in the event of a succesful request the response is JSON data in the following format with a status code of 200 (example): 
                { 
                success: true, 
                message: "Succesfully deleted phone with id 1"
            }
  • Error: in the event of an unsuccesful request the response is JSON data with the error object returned by sqlite with a status code of 500 (internal server error).
  • Input: The url query parameter "id" is required. See URL above for where to place it.

Create

Create and add one phone to the database

  • URL: "/create"
  • HTTP Method(s): POST
  • Success: in the event of a succesful request the response is JSON data in the following format with a status code of 200 (example): 
                { 
                success: true, 
                message: "Added phone"
            }
  • Error: in the event of an unsuccesful request the response is JSON data with the error object returned by sqlite with a status code of 500 (internal server error).
  • Input: No url query parameter is expected. Post data is expected - an object with the following required keys and a value of type: "brand" - string, "model" - string, "os" - string, "screensize" - number, and "image" - string.
